# Dota2 底层设计逻辑分析 - 视频计划

## 核心目标
从Dota2的所有数值数据出发，通过多维度分析和可视化，**抽象出一条贯穿游戏设计的底层逻辑链条**，用以解释游戏中的所有决策（英雄选择、装备路线、战略时机、资源分配等）。

## 第四部分：综合分析与底层逻辑

### 4.1 数据交叉分析

*英雄数值 × 技能数值*
- [ ] 英雄生命值曲线 vs 技能伤害曲线：不同阶段一套技能能打掉多少比例的血？
- [ ] 英雄生命值曲线 vs 普攻DPS曲线：不同阶段普攻多久能打死一个英雄？
- [ ] 英雄生命值曲线 vs 技能+普攻综合：一套技能+持续普攻，多久能击杀？
- [ ] 英雄魔法值 vs 技能消耗：各英雄能打几套技能？什么时候会"空蓝"？
- [ ] 英雄EHP vs 技能总伤害：几套技能能击杀一个英雄？
- [ ] 英雄理论DPS vs 技能DPS：物理输出和法术输出的效率对比
- [ ] 技能控制时间 vs 英雄攻击次数：一个控制内能打出多少伤害？

*英雄数值 × 世界数值*
- [ ] 英雄成长曲线 vs 经济曲线：什么时候经济能"喂饱"一个英雄？
- [ ] 英雄移速 vs 地图距离：不同英雄的"游走效率"差异
- [ ] 英雄清野速度 vs 野怪刷新间隔：谁能"吃满"野区？
- [ ] 英雄攻击距离 vs 地图关键位置：远程英雄的地形优势

*技能数值 × 世界数值*
- [ ] 技能冷却 vs 时间节点：技能CD和资源刷新如何同步？
- [ ] 技能伤害 vs 小兵/野怪血量：一个技能能清几个兵？能打几个野？
- [ ] 技能施法距离 vs 河道/高地宽度：技能能否跨越关键地形？
- [ ] 控制技能时间 vs 队友赶到时间：控住敌人后队友能否跟上？

*三者综合*
- [ ] 英雄成长 + 技能效率 + 经济收益 → 强势期判断
- [ ] 英雄EHP + 技能伤害 + 时间窗口 → 击杀窗口判断
- [ ] 英雄移速 + 技能CD + 地图距离 → 游走时机判断
- [ ] 英雄DPS + 技能DPS + 装备曲线 → 出装逻辑推导

*先手的价值*
- [ ] 先手的定义：什么是"先手"？（先手控制、先手伤害、先手位置）
- [ ] 先手技能分析：控制时间 × 后续伤害 = 先手价值
- [ ] 先手物品分析：跳刀/飞鞋/隐刀等机动性物品的"先手收益"
- [ ] 反先手分析：BKB/林肯/风杖等物品的"反先手价值"
- [ ] 先手与击杀窗口：先手后能否完成击杀？需要多少后续伤害？
- [ ] 先手与地图位置：视野、埋伏点、高地优势对先手的影响
